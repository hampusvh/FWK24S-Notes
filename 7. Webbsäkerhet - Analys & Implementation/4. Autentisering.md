# Autentisering

## Vad betyder autentisering?

Att **säkerställa att en person (eller ett system)** verkligen är den de utger sig för att vara.

→ En del av webbsäkerhetens *”AAA-modell”*: **Authentication**, **Authorization**, **Accounting**.

---
## Tre huvudkategorier av autentisering

1. **Något man KAN**

– t.ex. lösenord, PIN-kod, svar på hemlig fråga

→ Vanligast, men också mest sårbart

2. **Något man HAR**

– t.ex. ID-kort, säkerhetsdosa, mobiltelefon, certifikat

→ Exempel: BankID, YubiKey, engångskod via SMS

3. **Något man ÄR**

– biometri: fingeravtryck, ansikte, röst, iris

→ Svårt att förfalska, men kräver särskild hårdvara

---
## Multi-Factor Authentication (MFA / 2FA)

→ Kombination av minst två av ovanstående faktorer

Exempel:

- Lösenord (**kan**) + SMS-kod (**har**)

- Fingeravtryck (**är**) + lösenord (**kan**)

→ Ger starkare säkerhet, eftersom en angripare måste kompromettera flera faktorer.

---
## Länder och e-legitimation

Flera länder har system för **nationell e-identifiering**.

Exempel:

- **BankID** i Sverige, Norge, Finland

---
## CAPTCHA

**C**ompletely **A**utomated **P**ublic **T**uring test to tell **C**omputers and **H**umans **A**part

→ Skyddar mot automatiska inloggningar / bottar.

Vanliga varianter:

- **JCaptcha** (Java)

- **Ajax Fancy Captcha** (jQuery)

- **PHP CAPTCHA**

- **Simple Javascript CAPTCHA**

- **Captchas.net**, **Quick CAPTCHA**

---
## Olika autentiseringsmetoder i webbutveckling

### 1. **Basic Auth**

- Enkel HTTP-inbyggd metod

- Användarnamn + lösenord skickas (ofta Base64-enkodad)

- Kräver HTTPS för att vara säker

→ Lämplig endast för enklare tester eller interna system

### 2. **Databas-autentisering**

- Användaruppgifter lagras i databas

- Lösenord hashas (t.ex. med bcrypt, Argon2)

→ Standardmetod i webbtjänster

### 3. **OAuth / OAuth2 – Open Authentication**

- Standard för att låta användare logga in via tredje part

(t.ex. “Logga in med Google” eller “Logga in med GitHub”)

- Ger applikationer tillgång till vissa resurser utan att dela lösenord

→ OAuth2 = modernare version med fler flöden (Authorization Code Flow m.m.)

### 4. **JWT – JSON Web Token**

- Används för **session management**

- Klienten får en signerad token efter inloggning

- Token används för att identifiera användaren vid senare requests

→ Kan kombineras med:

- **Access token** (kort livstid)

- **Refresh token** (för att få nya access-tokens utan att logga in igen)

---

## Viktig princip

> “Man blir alltid autentiserad **som någon**.”

→ Systemet måste alltid veta **vilken identitet** en viss inloggning representerar.
